## 问题分析

### 为什么看不到加载的数据？

**根本原因：** Discover 页面组件根本没有被初始化和显示。

当前应用的主窗口 (`src/app.rs`) 只显示了一个固定的欢迎Label：

```rust
gtk::Label {
    set_label: "欢迎使用 Linn",
    ...
}
```

即使 Discover 组件的代码已经完整编写（包括API调用），但由于页面切换逻辑未实现，用户点击"发现音乐"时只是打印了日志，并没有切换到 Discover 页面。

### 解决方案

需要实现页面切换机制：

1. 在 `App` 中添加一个 `gtk::Stack` 来管理多个页面
2. 为每个导航项创建对应的页面组件
3. 根据导航按钮点击切换 Stack 的可见页面
4. 在页面首次显示时才初始化对应的组件
